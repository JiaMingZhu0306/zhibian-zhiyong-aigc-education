# 系统 UI 风格说明

AI 辅助生成，仅供教师参考。

## 一、整体定位

当前系统界面按“智慧教育信息系统 / 教师工作台”风格设计，服务于 16:9 演示录屏和比赛评审查看。页面不追求娱乐化效果，而强调清晰、可信、便于教师快速理解。

## 二、导航与布局

系统使用顶部文字导航，包含首页总览、作业文本分析、使用声明管理、班级分析看板、素养教育资源、隐私与使用边界等模块。导航按钮必须有文字，当前页面高亮，未选中页面使用浅色背景和浅边框，避免出现白字白底或默认黑框。

作业文本分析页面使用“文本输入 / 单张图片识别 / 批量图片导入”三个入口，三种输入方式最终进入同一套 AIGC 风险提示和看板同步流程。

## 三、视觉规范

- 主色：蓝色与少量紫色，用于按钮、高亮和重点信息。
- 背景：浅灰蓝或白色，避免大面积深色背景。
- 卡片：白底、圆角、浅边框、柔和阴影。
- 图表：使用 Plotly 浅色主题，避免黑底图、黑底表格和拥挤坐标轴。
- 表单：下拉框、输入框、文本框使用深色文字、浅色边框和清晰焦点状态。
- 风险结果：使用 AIGC 风险指数、风险等级、过程透明度和教师综合建议分区展示。

## 四、OCR 与批量导入页面

图片作业识别区域必须提示教师先遮挡学生姓名、班级、学号等可识别信息。OCR 识别可能存在漏字、错字或段落顺序错误，分析前需要教师确认或修正识别文本。系统不默认保存原图，看板只保存文本摘要、hash 和分析结果。

批量作业导入区域以文件列表、OCR 进度、可下载 CSV、批量分析结果表和同步状态为核心，适合展示“从多张图片到班级看板”的完整流程。

## 五、录屏建议

录制演示视频时建议使用 1920×1080 分辨率，浏览器缩放保持 100%。优先展示首页 Hero、作业文本分析、单张图片识别、批量作业导入、班级分析看板和隐私边界页面。录屏时使用匿名演示样例，不展示真实学生姓名、学校、班级、联系方式或完整原文隐私信息。

## 六、边界文案

所有页面应保留以下边界：AIGC 风险指数表示模型参考概率，不代表文本中 AI 生成内容占比，不作为学生违纪判定或处分依据。系统输出仅供教师参考，真实课堂应用需要结合草稿、访谈、学生 AI 使用声明和教师过程性观察综合判断。
